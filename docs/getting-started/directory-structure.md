# Directory Structure

After running `composer create-project webrium/webrium my-app`, you will have the following structure:

```
my-app/
├── app/
│   ├── Config/           ← Application config files (DB.php, Mail.php, ...)
│   ├── Controllers/      ← Controller classes
│   ├── Helpers/          ← Custom helper functions
│   ├── Middleware/       ← Middleware classes
│   ├── Models/           ← Eloquent model classes
│   ├── Routes/           ← Route definition files
│   ├── Services/         ← Service / business logic classes
│   └── Views/            ← Template files (.php)
│
├── public/
│   ├── index.php         ← Entry point — all requests go through here
│   ├── assets/           ← Compiled JS/CSS (generated by Vite)
│   └── uploads/          ← User-uploaded files
│
├── resources/
│   └── js/
│       └── app.js        ← Vite entry point
│
├── storage/
│   ├── Framework/
│   │   ├── Cache/        ← View cache and app cache
│   │   └── Sessions/     ← Session files
│   ├── Langs/            ← Language/translation files
│   └── Logs/             ← Application log files
│
├── vendor/               ← Composer dependencies (auto-generated)
├── .env                  ← Environment variables (never commit to Git)
├── .env.example          ← Example env file (commit this)
├── composer.json
├── package.json
└── vite.config.js
```

## Key Directories Explained

### `app/`

This is where your application code lives.

- **`Controllers/`** — Classes that handle HTTP requests and return responses.
- **`Models/`** — FoxDB Eloquent models, each representing a database table.
- **`Routes/`** — Route files loaded by `Route::source()` in `public/index.php`.
- **`Middleware/`** — Classes that intercept and filter HTTP requests.
- **`Config/`** — Configuration files loaded at startup (database, mail, etc.).
- **`Views/`** — PHP template files rendered by the View engine.

### `public/`

The web server document root. Only `public/` should be publicly accessible.

- **`index.php`** — The front controller. All requests are routed through this file.
- **`assets/`** — Compiled and versioned frontend files output by Vite.

### `storage/`

Files generated at runtime. This directory should be writable by the web server.

- **`Sessions/`** — PHP session files.
- **`Cache/`** — Compiled view templates and static HTML cache.
- **`Logs/`** — Log files managed by the Console `log` command.
- **`Langs/`** — Translation files for localization.

### `resources/`

Source files for the frontend build pipeline.

## The Entry Point

All HTTP requests are routed through `public/index.php`. A typical entry point looks like:

```php
<?php
require __DIR__ . '/../vendor/autoload.php';

use Webrium\App;
use Webrium\Route;
use Webrium\File;

App::initialize(__DIR__ . '/..');

File::source('config', ['Config.php', 'DB.php']);

Route::source(['web.php', 'api.php']);

App::run();
```
